### Ancestry Person-County Scraper

Scrapes individual person records from Ancestry Library Edition to find their residential county. Uses Selenium to authenticate via a university library proxy, search the 1790 census collection, and parse county information from results.

### Usage

```bash
python main.py CT --year 1790
```

Requires Ancestry Library Edition access via a university proxy. See `ancestry_scraper/auth.py` for proxy configuration.

### Code

- `ancestry_scraper/` — Core package (auth, config, parser, search, session, storage, worker)
- `main.py` — CLI entrypoint. Reads lookup CSVs, calls `worker.process_name()` for each row
- `create_names_lookup.py` — Reads cleaned loan office certificates, deduplicates by name+state, writes lookup CSVs
- `analyze_results.py` — Loads scraper results, parses county strings, generates choropleth maps
- `script.py` — Interactive search utility for debugging individual names
- `tests/` — Unit and integration tests (`pytest`)

### Input

- `output/scrape/ancestry_person_county_scraper/data/loan_office_certificates_cleaned.csv`
- `output/scrape/ancestry_person_county_scraper/names_to_lookup/names_to_lookup_{state}.csv`

### Output

All output in `output/scrape/ancestry_person_county_scraper/`:
- `results/results_{state}.csv` — Scraped county results per state
- `progress/progress_{state}.json` — Checkpoint files for resume
- `maps/*.png` — Choropleth maps generated by `analyze_results.py`
